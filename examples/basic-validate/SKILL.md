# Code Review

## Instructions

1. Read the provided code diff
2. Analyze for bugs, style issues, and potential improvements
3. Return a structured ReviewReport with findings

## Examples

**Input**: A diff from a pull request adding a new API endpoint
**Output**: ReviewReport with severity-tagged findings
